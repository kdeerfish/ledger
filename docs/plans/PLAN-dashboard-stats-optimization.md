# Dashboard & 统计页面优化需求

## 需求 1：统计卡片点击不跳转

### 现状
- Dashboard 的收入/支出/结余卡片没有跳转（✅ 已满足）
- Stats 页面的"总收入""总支出""结余""笔数"卡片点击会跳转到 `/transactions`

### 需求
- 统计卡片（总收入、总支出、结余、笔数）只用于展示汇总数据
- 点击不跳转，不触发任何操作
- 保持卡片的视觉样式，去掉 `cursor: pointer`

### 改动
- `Stats.jsx`：`StatCard` 组件去掉 `onClick` 和 `cursor: pointer`

## 需求 2：增加图表样式

### 现状
- Stats 页面已有：环形图、饼图、柱状图（3种）
- Dashboard 只有：柱状图（月度）、折线图（累计）、环形图（类别）

### 需求
在 Stats 页面增加图表类型选项：

| 新增类型 | 说明 | 适用场景 |
|---------|------|---------|
| **水平柱状图** | `horizontalBar` | 类别名称较长时更好展示 |
| **堆叠柱状图** | 多个月份的类别堆叠 | 看各类别月度变化 |
| **折线图** | 按月份的趋势折线 | 看某个维度的时间趋势 |

优先做：**水平柱状图**（改动最小，直接加 `indexAxis: 'y'`）

### 改动
- `Stats.jsx`：`CHART_OPTIONS` 增加 `horizontalBar` 选项
- 图表渲染逻辑加对应分支

## 需求 3：图表点击不跳转，直接在当前页面展示明细

### 现状
- 点击图表（饼图/柱状图）或数据表格行 → 跳转到 `/transactions?category=xxx`
- 用户被迫离开当前页面，丢失上下文

### 需求
点击图表或表格行时：
1. **不跳转**，停留在统计页面
2. 在数据明细区域**展开/高亮**该类目的详细交易列表
3. 显示：该类目下每笔交易的日期、金额、商家、备注
4. 提供"查看全部"链接（可选跳转到交易页）

### 交互设计

```
当前布局：
  [图表区]  [数据明细表格]

点击"餐饮美食"后：
  [图表区]  [数据明细表格]
                     ↓
            [餐饮美食 - 明细展开区]
            ├ 2024-06-15  星巴克  ¥35.00
            ├ 2024-06-14  美团外卖 ¥42.50
            ├ 2024-06-13  食堂    ¥15.00
            └ ... 共45笔，合计 ¥2,340
            [收起]  [查看全部 →]
```

### 数据来源
- 点击时调用 `GET /api/transactions?category=xxx&limit=50`
- 或者在页面加载时预取所有数据，点击时前端过滤（数据量不大时）

### 改动
- `Stats.jsx`：
  - 新增 `selectedGroup` state
  - 新增 `detailTransactions` state
  - 图表点击改为展开明细区域（不跳转）
  - 表格行点击改为展开/收起明细
  - 新增明细列表组件（可折叠）
- `Dashboard.jsx`：
  - 同样的逻辑，饼图点击不跳转，改为展开明细

## 需求 4：大额交易导致图表比例失衡优化

### 现状
- 如果有一笔 50 万的收入，其他几千块的收入在图表上几乎看不到
- 饼图/柱状图被大额项主导，小额项被挤压

### 需求
多种优化策略结合：

#### 策略 A：对数刻度（柱状图）
- Y 轴使用对数刻度（`logarithmic`），让大额和小额都能看到
- 缺点：对数刻度不直观，用户可能看不懂
- **不作为默认，作为可切换选项**

#### 策略 B：分组合并（饼图/柱状图）
- 占比小于 3% 的项自动合并为"其他"
- 点击"其他"可展开查看所有小额项
- **推荐作为默认行为**

#### 策略 C：双轴 / 双图表
- 大额和小额分开显示（上下两个图表）
- 复杂度高，不推荐

#### 策略 D：缩放 + 悬浮详情
- 柱状图支持拖拽缩放（Chart.js zoom 插件）
- 饼图/环形图点击某块后放大显示
- 中等复杂度

#### 最终方案（推荐）

**饼图/环形图**：
- 占比 < 3% 的项合并为"其他"
- "其他"可点击展开
- 展开后显示完整列表

**柱状图**：
- 默认普通刻度
- 增加"对数刻度"切换按钮
- 或使用 Chart.js 的 `suggestedMax` 让最大值不完全主导

**数据明细表格**：
- 已经有占比进度条，这个不受影响
- 进度条加上数值标签，小额也能看到具体金额

### 改动
- `Stats.jsx` + `Dashboard.jsx`：
  - 饼图数据预处理：小占比项合并
  - 柱状图增加刻度切换选项
  - 进度条优化

## 需求 5：交易排除功能

### 概述
大额一次性交易（如买房、买车、大额转账）会影响统计图表的准确性。
用户需要能"排除"这些交易，让日常统计数据更有参考价值。

### 数据设计

使用现有的标签机制：
- 内置一个特殊标签 `排除统计`（系统标签，不可删除）
- 被打上此标签的交易在统计/图表中默认不显示
- 提供全局开关：是否包含被排除的交易

### 排除方式

#### 方式 1：手动排除（网页）
- 交易列表：每条交易加一个"排除"按钮（眼睛图标 👁）
- 点击后交易变灰 + 打上"排除统计"标签
- 再次点击恢复
- 交易详情弹窗：也加一个排除开关

#### 方式 2：导入时自动检测
- 导入引擎检测异常大额交易
- 判断标准：金额 > 该用户历史数据中同类别的 **5 倍平均值** 或 **P99 分位数**
- 导入预览时高亮提醒：
  ```
  ⚠️ 发现 2 笔异常大额交易：
    1. 2024-06-15 购房支出 ¥500,000（类别"住房"平均 ¥8,000）
    2. 2024-06-20 转账收入 ¥200,000（类别"转账"平均 ¥5,000）
  是否排除这些交易的统计影响？[排除] [保留] [逐条确认]
  ```

### 统计页面联动

- 统计页面顶部增加一个 toggle：`☑ 排除标记交易`
  - 默认勾选 = 排除被标记的交易（日常统计）
  - 取消勾选 = 包含所有交易（完整统计）
- 被排除的交易在图表上不显示，在表格中灰色显示
- Dashboard 同样联动

### 改动

**数据库**：
- 无需改表，用现有 tags 机制

**后端**：
- `GET /api/transactions`：增加 `exclude_tagged` 参数（默认 false）
- `GET /api/stats`、`GET /api/summary`、`GET /api/trends`：增加 `exclude_tagged` 参数
- 导入引擎：增加大额交易检测逻辑

**前端**：
- `Transactions.jsx`：每行加排除按钮
- `Stats.jsx`：顶部加排除 toggle
- `Dashboard.jsx`：顶部加排除 toggle
- `Import.jsx`：Step 3 增加大额检测提醒
