# 智能导入工作流

## 场景

用户有 CSV 格式的记账数据（来自支付宝、微信、随手记、银行等），需要导入到系统中。

## 完整流程

### 步骤 1：用户发起导入

**用户说**：
> 导入支付宝账单

**Agent 操作**：
1. 让用户上传 CSV 文件（通过 Web 页面 `/import` 或提供文件路径）
2. 如果是 Web 端，直接上传文件
3. 如果是 CLI/Docker，让用户把文件放到数据目录

### 步骤 2：分析文件，预览映射

**Agent 执行**（Web 端用户直接在页面操作）：

```bash
curl -X POST $BASE_URL/api/import/preview \
  -F "file=@/path/to/alipay_2024.csv"
```

**Agent 解读返回结果**：
- `detected_source`：自动识别出"支付宝"
- `mapping`：已自动映射的列（如"交易对方"→merchant，"支付金额"→amount）
- `unmapped_columns`：未映射的列（如"交易单号"→存入 extra_data）
- `suggested_tags`：建议标签（如"支付宝导入"、"2024-06"）
- `duplicate_estimate`：预估重复数

### 步骤 3：确认映射，执行导入

**Agent 展示预览给用户确认**：
> 检测到支付宝格式，共 128 条记录：
> - 已自动映射：交易时间→日期，支付金额→金额，交易对方→商家
> - 未映射（保留为其他信息）：交易单号、商户单号
> - 预计 3 条重复将跳过
> - 建议标签：支付宝导入、2024-06
> 
> 确认导入？

**用户确认后，Agent 执行**：

```bash
curl -X POST $BASE_URL/api/import/execute \
  -F "file=@/path/to/alipay_2024.csv" \
  -F 'mapping={"交易创建时间":"date","支付金额":"amount","交易对方":"merchant","商品说明":"note","收/付款方式":"account","收/支":"type"}' \
  -F 'tags=["支付宝导入","2024-06"]' \
  -F "skip_duplicates=true" \
  -F "source=支付宝"
```

### 步骤 4：确认结果

**Agent 报告**：
> 导入完成！
> - 成功导入：125 条
> - 跳过（无效）：0 条
> - 跳过（重复）：3 条
> - 标签：支付宝导入、2024-06

### 步骤 5：学习数据（可选）

**Agent 建议**：
> 建议执行"学习记账"来分析导入的数据，这样我能记住你的账户、商家、类别习惯。

```bash
curl $BASE_URL/api/analyze
```

## 各平台 CSV 列名对照

导入引擎自动识别以下列名，无需手动映射：

| 系统字段 | 支付宝 | 微信 | 随手记 | 银行 |
|---------|--------|------|--------|------|
| type | 收/支 | 收支类型 | 交易类型 | — |
| amount | 金额 | 金额 | 金额 | 交易金额 |
| date | 交易创建时间 | 交易时间 | 日期 | 交易日期 |
| category | — | — | 类别 | — |
| merchant | 交易对方 | 商品 | 商家 | 交易对方 |
| account | 收/付款方式 | 支付方式 | 账户 | — |
| note | 商品说明 | 商品名称 | 备注 | 交易摘要 |

未匹配的列值自动存入 `extra_data` 字段，不丢失。

## 注意事项

1. 编码自动检测（UTF-8/GBK/GB2312），无需手动转换
2. 文件大小限制 10MB
3. 重复检测基于：同日+同类型+同金额+同类别
4. 导入后可通过 `/export` 导出兼容 CSV，实现往返不丢数据
