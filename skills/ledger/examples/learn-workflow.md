# 学习工作流示例

## 场景

用户导入数据后，需要让 Agent 学习用户的记账习惯。

## 完整流程

### 步骤 1：用户发起学习

**用户说**：
> 学习

### 步骤 2：Agent 执行分析

**Agent 执行**：
```bash
curl http://127.0.0.1:5800/api/analyze
```

**输出示例**：
```
【总览】共 4128 笔记录，总金额 991792.23 元

【账户】（= 付款方式/资金来源）
  信用卡: 1484笔 73924.85元
  微信零钱: 844笔 110630.03元
  招商银行: 273笔 521550.65元

【商家】（= 消费场所/平台）
  拼多多: 771笔 17579.85元
  京东: 236笔 25262.66元
  美团: 235笔 7404.97元

【类别→子类别】
  食品酒水:
    - 早午晚餐 (618笔 13017.33元)
    - 零食 (106笔 3244.30元)

【成员】
  本人: 2942笔 679919.94元
  家庭公用: 609笔 254853.43元
```

### 步骤 3：Agent 保存记忆

**Agent 用 remember 保存**：
- 常用账户：信用卡、微信零钱、招商银行
- 常用商家：拼多多、京东、美团
- 家庭成员：本人、fish、家庭公用

### 步骤 4：确认完成

**Agent 回复**：
> 已学习完成！以后记账时我会自动参考这些习惯。

## 学习后的应用

### 场景 1：记账

**用户说**：
> 京东买了200块零食

**Agent 根据学习结果填写**：
```bash
curl -X POST http://127.0.0.1:5800/api/transactions \
  -H 'Content-Type: application/json' \
  -d '{"type":"支出","amount":200,"category":"食品酒水","subcategory":"零食","merchant":"京东","account":"信用卡"}'
```

### 场景 2：不确定时

**用户说**：
> 拼多多买了一双鞋

**Agent 回复**：
> 金额是多少？用哪个账户支付的？（常用：信用卡、微信零钱）

## 注意事项

1. 学习后记忆永久生效，无需重复学习
2. 如果数据变化较大，可以重新学习
3. 学习结果会自动应用到后续记账中
