# 学习工作流示例

## 场景

用户导入数据后，需要让 Agent 学习用户的记账习惯。

## 完整流程

### 步骤 1：用户发起学习

**用户说**：
> 学习

### 步骤 2：Agent 执行分析

**Agent 执行**：
```bash
python3 scripts/ledger_cli.py analyze '{}'
```

**输出示例**：
```
═══════════════════════════════════════════════════
📊 用户数据分析报告（供 agent 学习用）
═══════════════════════════════════════════════════

【总览】共 4128 笔记录，总金额 991792.23 元

【账户】（= 付款方式/资金来源，钱从哪里扣或收到哪里）
  信用卡: 1484笔 73924.85元
  微信零钱: 844笔 110630.03元
  招商银行: 273笔 521550.65元

【商家】（= 消费场所/平台，在哪里花的钱）
  拼多多: 771笔 17579.85元
  京东: 236笔 25262.66元
  美团: 235笔 7404.97元

【类别→子类别】（= 花在什么类型的东西上）
  食品酒水:
    - 早午晚餐 (618笔 13017.33元)
    - 零食 (106笔 3244.30元)
  居家物业:
    - 日常用品 (428笔 8389.91元)
    - 房租水电 (90笔 131208.87元)

【成员】（= 谁花了这笔钱/谁收到这笔钱）
  本人: 2942笔 679919.94元
  家庭公用: 609笔 254853.43元
  fish: 229笔 7709.14元
═══════════════════════════════════════════════════
```

### 步骤 3：Agent 保存记忆

**Agent 用 remember 保存**：

```markdown
# 用户记账习惯

## 账户（= 付款方式/资金来源）
- 信用卡: 1484笔（常用主力账户）
- 微信零钱: 844笔（日常小额支付）
- 招商银行: 273笔（大额收入/转账）

## 商家（= 消费场所/平台）
- 拼多多: 771笔（最常用网购平台）
- 京东: 236笔（网购，品类较全）
- 美团: 235笔（外卖/团购）

## 类别→子类别
- 食品酒水: 早午晚餐/零食/奶茶/饮料/水果
- 居家物业: 日常用品/房租水电/维修保养

## 成员
- 本人: 2942笔
- 家庭公用: 609笔
- fish: 229笔
```

### 步骤 4：确认完成

**Agent 回复**：
> 已学习完成！我已经记住你的习惯：
> 
> **常用账户**：信用卡、微信零钱、招商银行
> **常用商家**：拼多多、京东、美团
> **家庭成员**：本人、fish、家庭公用
> 
> 以后记账时我会自动参考这些习惯。例如：
> - 说"京东买了零食"→ merchant=京东, category=食品酒水
> - 说"微信打车"→ account=微信零钱, merchant=滴滴

## 学习后的应用

### 场景 1：记账

**用户说**：
> 京东买了200块零食

**Agent 根据学习结果填写**：
```bash
python3 scripts/ledger_cli.py add '{"type":"支出","amount":200,"category":"食品酒水","subcategory":"零食","merchant":"京东","account":"信用卡"}'
```

### 场景 2：不确定时

**用户说**：
> 拼多多买了一双鞋

**Agent 回复**：
> 已记录：拼多多买了鞋，金额是多少？
> 
> 另外，请问：
> 1. 用哪个账户支付的？（常用：信用卡、微信零钱）
> 2. 记在哪个成员名下？（常用：本人、fish）

## 注意事项

1. 学习后记忆永久生效，无需重复学习
2. 如果数据变化较大，可以重新学习
3. 学习结果会自动应用到后续记账中
