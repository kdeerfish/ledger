# 🌐 Web 界面使用手册

启动后浏览器打开 `http://localhost:5800`（或你的 NAS/服务器 IP:5800）即可使用。

界面采用 **React + Vite + Chart.js**，响应式设计，手机 / 平板 / 电脑均可用。

---

## 📑 页面总览

| 页面 | 路径 | 主要功能 |
|------|------|---------|
| **概览** | `/` | 收支卡片 + 月度柱状图 + 累计折线图 + 类别环形图 |
| **交易** | `/transactions` | 多维筛选表格 + 分页 + 编辑 / 删除 |
| **记一笔** | `/add` | 模板选择 + 字段自动建议 + 子类别 + 标签选择器 |
| **预算** | `/budgets` | 总览卡片 + 类别进度条 + 超支预警 + 执行明细 |
| **类别 + 标签** | `/categories` | 类别层级统计 + 标签管理（颜色、删除） |
| **统计** | `/stats` | 9 种分组 × 3 种图表，点击图表跳转筛选 |

---

## 1. 概览 Dashboard

打开默认看到的第一页，**最重要的指标一眼看完**：

- 📊 **收支卡片**：总收入 / 总支出 / 结余 / 日均支出
- 📈 **月度柱状图**：每个月收支对比
- 📉 **累计折线图**：累计收入 / 累计支出趋势
- 🍩 **类别环形图**：各类别支出占比，**点击任一区块自动跳转到交易页并筛选该类别**
- 📋 **最近交易**：点击查看详情

---

## 2. 交易管理（最常用）

### 2.1 筛选交易

顶部筛选栏支持多维组合筛选：

| 筛选条件 | 说明 |
|---------|------|
| 关键词 | 搜索备注 / 类别 / 商家 |
| 类型 | 支出 / 收入 |
| 类别 | 一级类别（点击展开选子类别） |
| 账户 | 微信 / 支付宝 / 银行卡等 |
| 标签 | 多选标签 |
| 日期范围 | 起止日期 |

所有筛选条件**组合生效**，比如："2026 年 6 月 + 支出 + 食品 + 标签『外卖』"。

### 2.2 记一笔

点右上角 **「记一笔」** 按钮，或去 `/add` 页面：

1. **选模板**（顶部）：常用模板点击直接带入字段
2. **选类型**：支出 / 收入
3. **填金额**
4. **选类别**：支持搜索；选完一级类别会自动展开子类别
5. **选账户 / 商家 / 标签**（可选）：系统会自动建议（基于历史输入）
6. **填备注**（可选）
7. **保存**

> 💡 系统会自动检查重复（同类目同金额同日期），如需强制保存点 **「强制保存」**。

### 2.3 编辑 / 删除

- **编辑**：点表格行右侧的编辑图标
- **删除**：点删除图标（**软删除**，可恢复）
- **硬删除**：在回收站里点「永久删除」

## 3. 预算管理

### 3.1 设置预算

去 `/budgets` 页面，点 **「设置预算」**：

| 字段 | 说明 |
|------|------|
| 年 / 月 | 这个预算生效的月份 |
| 维度 | 按 **类别 / 账户 / 成员 / 项目** 之一 |
| 值 | 比如类别维度填 "食品"，账户维度填 "微信" |
| 金额 | 限额 |

### 3.2 预算模板（推荐）

如果你每个月预算差不多，可以用 **预算模板** 一次创建，下个月一键应用：

1. 设好当月预算
2. 点 **「保存为模板」**
3. 下个月初点 **「应用模板到本月」**

系统还有 **智能推荐**：根据历史消费自动建议预算值。

### 3.3 超支预警

预算卡片进度条变红 = 已用 ≥ 100%。首页 Dashboard 也会显示超支提醒。

## 4. 类别 + 标签管理

### 4.1 类别

- **类别** 是层级结构（食品 → 早餐 / 午餐 / 晚餐 / 零食）
- Web 界面只展示，**新建类别** 在记账时直接输入即可，系统自动建
- 子类别同理

### 4.2 标签

标签是**平级的、跨类别的**，用于灵活标记：

- 例：一笔餐饮支出，可以同时打「工作日」+ 「外卖」+ 「公司附近」3 个标签
- **创建标签**：去 `/categories` 页面，「标签」卡片 → 输入名字 + 选颜色
- **删除标签**：删除后会自动从所有交易上移除

---

## 5. 统计图表

`/stats` 页面是分析利器：

- **9 种分组维度**：类别 / 子类别 / 账户 / 商家 / 项目 / 成员 / 月 / 标签 / 类型
- **3 种图表**：环形图 / 饼图 / 柱状图
- 收入 / 支出 **双图展示**
- **点击图表任一块 → 自动跳转到交易页并带入筛选**

### 常用分析场景

| 你想看什么 | 怎么操作 |
|-----------|---------|
| 这个月钱花在哪了 | 统计 → 按「类别」分组 → 支出 |
| 谁是常去的商家 | 统计 → 按「商家」分组 → 支出 |
| 各账户用了多少 | 统计 → 按「账户」分组 |
| 某标签花了多少 | 统计 → 按「标签」分组 |
| 月度趋势 | 统计 → 按「月」分组 |

## 6. 导入 / 导出

### 6.1 导入 CSV

支持随手记（MyMoney）导出的 CSV 格式：

```bash
docker exec -it ledger python scripts/import_ledger.py /data/your-file.csv
```

**自动去重**（同类目同金额同日期会跳过），导入完成后会显示统计。

### 6.2 导出

- Web 界面：交易页右上角「导出」按钮（CSV / JSON）
- CLI：`python scripts/cli.py export --output report.csv`

## 7. 移动端使用

界面响应式，手机浏览器直接访问 NAS IP 即可：

- **iOS Safari**：「分享」→「添加到主屏幕」，像 App 一样用
- **Android Chrome**：菜单 →「添加到主屏幕」

进阶：可以用 Tailscale / 自建 VPN 让手机在外面也能访问家里的 NAS。

---

## 8. 多人使用

**当前版本是单用户设计**，不内置登录认证。多人场景建议：

| 方案 | 复杂度 | 隔离度 |
|------|--------|--------|
| 每人一个容器实例（不同端口） | 低 | 高 |
| 部署在 NAS，多人共享一个账本 | 低 | 低（看得到所有人的账） |
| NAS + 反向代理 + BasicAuth | 中 | 中 |
| 多用户版（未来路线图） | — | 高 |
